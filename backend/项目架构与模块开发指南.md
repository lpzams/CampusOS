# CampusOS 后端项目架构与模块开发指南

## 1. 项目整体认识

这个仓库是一个使用 Spring Boot 3、Java 17、Maven 和 MyBatis-Plus 开发的单体后端。

它虽然拆成了 5 个 Maven 模块，但最终只会启动一个 Spring Boot 服务，并不是微服务。

整体调用关系如下：

```text
前端 / 小程序
      │ HTTP 请求
      ▼
campus-api                 接收请求、登录校验、返回结果
      │
      ▼
campus-application         编排业务流程
      │
      ▼
campus-domain              业务对象、业务规则、仓储接口
      ▲
      │ 实现 domain 定义的接口
campus-infrastructure      MyBatis、MySQL、JWT 等具体技术
      │
      ▼
    MySQL

campus-common              各层共同使用的 Result、分页和异常
```

所有 HTTP 接口已按业务模块拆分 Controller 和 AppService。成绩、支付、校园卡、
报修、商品、活动、地图等核心规则位于 `campus-domain` 的纯 Java 领域对象中；
`CampusRecord + JSON` 仅作为当前 infrastructure 的存储实现，Controller 不直接接触它。

### 2.1 `.idea/`

IDEA 自动生成的项目配置，可能包含：

- 编辑器状态

它不属于业务代码，一般不需要手动修改。

### 2.2 根目录 `pom.xml`

根 Maven 配置负责管理整个项目，并声明 5 个子模块：

```xml
<modules>
    <module>campus-common</module>
    <module>campus-domain</module>
    <module>campus-application</module>
    <module>campus-infrastructure</module>
    <module>campus-api</module>

```

它还统一管理：

- Java 17
- Spring Boot 3.2.5
- MyBatis-Plus
- JWT
- Hutool
- 内部模块版本

根 `pom.xml` 自己不会生成可运行程序，它是整个 Maven 工程的总配置。

### 2.3 `docker/`

```text

```

这个配置主要供 Docker 构建使用，通过阿里云 Maven 镜像下载依赖。

### 2.4 `Dockerfile`

Dockerfile 使用多阶段构建：

```text

```

最终容器中的启动命令是：


## 3. Java 标准目录说明

每个 Maven 模块基本使用下面的目录结构：

```text
模块名/
├── pom.xml
└── src/
    └── main/
        ├── java/
        │   └── com/campus/...
        └── resources/
```

含义如下：

| 目录 | 作用 |
|---|---|
| `src` | 源代码目录 |
| `src/main` | 正式运行代码 |
| `src/main/java` | Java 源代码 |
| `src/main/resources` | 配置、Mapper XML、静态资源等 |
| `com/campus` | Java 包名目录 |

通常还应该有 `src/test/java` 用来编写测试，但当前仓库没有测试目录。

## 4. `campus-common`：公共基础模块

```text
campus-common/
├── pom.xml
└── src/main/java/com/campus/common/
    ├── api/
    │   ├── Result.java
    │   ├── ResultCode.java
    │   └── PageResult.java
    ├── exception/
    │   └── BusinessException.java
    └── query/
        └── PageQuery.java
```

该模块不负责具体校园业务，只提供各模块共同使用的基础对象。

### 4.1 `common/api/Result.java`

统一接口返回结构：

```json
{
  "code": 0,
  "msg": "操作成功",
  "data": {}
}
```

Controller 应统一使用：

```java
Result.success(data);
Result.fail("操作失败");
```

### 4.2 `common/api/ResultCode.java`

统一管理成功、失败、参数错误、系统错误等状态码。

### 4.3 `common/api/PageResult.java`

统一分页返回格式：

```json
{
  "pageNum": 1,
  "pageSize": 10,
  "total": 38,
  "pages": 4,
  "list": []
}
```

### 4.4 `common/exception/BusinessException.java`

表示可预期的业务错误，例如：

```java
throw new BusinessException(404, "新闻不存在");
```

异常最终由 API 层统一处理。

### 4.5 `common/query/PageQuery.java`

所有分页查询的父类，提供：

```java
pageNum
pageSize
```

它还会限制 `pageSize` 最大为 100，避免一次查询过多数据。

## 5. `campus-domain`：领域核心模块

```text
campus-domain/
├── pom.xml
└── src/main/java/com/campus/domain/
    ├── auth/
    │   └── CredentialService.java
    ├── news/
    │   ├── News.java
    │   └── NewsRepository.java
    └── shared/
        ├── CampusRecord.java
        └── CampusRecordRepository.java
```

这一层表达“业务是什么”。

它不应该依赖或了解：

- HTTP
- Controller
- MyBatis
- MySQL
- JWT 的具体库
- Spring MVC

### 5.1 `domain/news/News.java`

`News` 是新闻领域实体。它除了保存数据，还包含业务行为：

```java
news.publish();
news.offline();
news.increaseView();
```

例如发布新闻：

```java
public void publish() {
    if (title == null || title.isBlank()) {
        throw new IllegalStateException("标题不能为空");
    }

    status = Status.PUBLISHED;
    publishTime = LocalDateTime.now();
}
```

### 5.2 `domain/news/NewsRepository.java`

新闻仓储接口只声明业务需要什么存储能力：

```java
News save(News news);
Optional<News> findById(Long id);
void deleteById(Long id);
```

它不写 MyBatis 代码，也不知道数据最终存在哪里。

### 5.3 `domain/auth/CredentialService.java`

声明认证功能需要的能力：

```java
String hash(String password);
boolean matches(String password, String hash);
String createToken(Long userId, String username, Integer userType);
TokenClaims parseToken(String token);
```

具体的 BCrypt 和 JWT 实现放在 infrastructure 模块。

### 5.4 `domain/shared/`

这部分是项目的通用业务数据模型。

`CampusRecord` 的结构近似为：

```java
CampusRecord {
    Long id;
    String type;
    Map<String, Object> data;
}
```

例如一条课程数据可以表示为：

```text
type = "course"
data = {
    courseName: "高等数学",
    teacher: "张老师"
}
```

当前通知、课程、成绩、缴费、活动等大量功能都使用这个通用模型。

### 6.1 `application/news/command/`

Command 表示会修改系统状态的请求，例如创建、修改和删除。

`CreateNewsCommand` 表示“创建新闻”：

```java
@NotBlank
private String title;

@NotBlank
private String content;
```

它负责接收前端 JSON，并进行基本格式校验。

### 6.2 `application/news/query/`

Query 表示查询条件。

`NewsPageQuery` 包含：

```java
pageNum
pageSize
keyword
category
```

对应请求：

```http
GET /api/news?pageNum=1&pageSize=10&keyword=奖学金
```

### 6.3 `application/news/dto/`

DTO 是返回给 Controller 和前端的数据结构。

不要混淆下面三个对象：

| 对象 | 所在层 | 用途 |
|---|---|---|
| `News` | domain | 表达业务和业务规则 |
| `NewsDTO` | application | 返回给前端 |
| `NewsPO` | infrastructure | 映射数据库表 |

### 6.4 `application/news/NewsAppService.java`

新闻用例的总入口。

例如查看详情：

```java
@Transactional
public NewsDTO getDetail(Long id) {
    News news = newsRepository.findById(id)
        .orElseThrow(() -> new BusinessException("新闻不存在"));

    news.increaseView();
    newsRepository.save(news);

    return toDTO(news);
}
```

它组织的是：

```text
查询 → 执行业务动作 → 保存 → 转换 DTO
```

### 6.5 `application/auth/AuthAppService.java`

负责：

- 用户名密码登录
- 短信验证码登录
- 微信模拟登录
- 注册
- 重置密码
- 修改密码
- 生成和刷新 token
- 查询和修改个人资料

### 6.6 `application/shared/CampusAppService.java`

这是通用 CRUD 服务，可以处理任意 `type`：

```java
app.list("course");
app.create("repair", body);
app.page("activity", page, size);
```

这种写法适合快速原型。正式扩展复杂功能时，建议按照新闻模块创建强类型结构。

### 7.1 `infrastructure/auth/JwtCredentialService.java`

它实现 domain 定义的 `CredentialService`：

```java
public class JwtCredentialService implements CredentialService
```

内部具体使用：

- BCrypt 加密和验证密码
- JJWT 生成 token
- JJWT 解析 token

依赖关系是：

```text
domain 定义接口
        ▲
        │ implements
infrastructure 提供实现
```

### 7.2 `infrastructure/persistence/news/NewsPO.java`

PO 是数据库对象，对应 `t_news` 表：

```java
@TableName("t_news")
public class NewsPO
```

### 7.3 `infrastructure/persistence/news/NewsMapper.java`

```java
public interface NewsMapper extends BaseMapper<NewsPO>
```

继承 MyBatis-Plus 的 `BaseMapper` 后会获得：

```java
insert()
selectById()
updateById()
deleteById()
selectList()
```

### 7.4 `infrastructure/persistence/news/NewsConverter.java`

负责转换：

```text
NewsPO  ⇄  News
数据库      业务
```

数据库结构变化时，主要修改 PO 和 Converter，不让数据库细节进入 domain。

### 7.5 `infrastructure/persistence/news/NewsRepositoryImpl.java`

实现 domain 层的 `NewsRepository`：

```java
@Repository
public class NewsRepositoryImpl implements NewsRepository
```

内部调用 `NewsMapper` 操作数据库。

### 7.6 `infrastructure/persistence/shared/`

负责把通用 `CampusRecord` 存入 `t_campus_record`。

`recordData` 会被序列化成 JSON 字符串：

```text
CampusRecord
      │ ObjectMapper
      ▼
CampusRecordPO
      │ MyBatis
      ▼
t_campus_record
```

### 8.1 `CampusApplication.java`

这是整个后端唯一的启动入口：

```java
@SpringBootApplication(scanBasePackages = "com.campus")
@MapperScan("com.campus.infrastructure.persistence")
```

- `@SpringBootApplication` 扫描 Controller、Service 和 Repository。
- `@MapperScan` 扫描 MyBatis Mapper。

启动时还会在没有用户数据的情况下创建默认管理员：

```text
用户名：admin
密码：123456
```

### 8.2 `api/auth/`

#### `Authenticated.java`

接口加上 `@Authenticated` 表示需要登录。

#### `AuthInterceptor.java`

请求进入 Controller 前检查：

```http
Authorization: Bearer <JWT>
```

没有 token 或 token 过期时抛出 401。

#### `AuthSession.java`

只保存当前请求的登录用户信息，主要提供：

```java
session.userId();
session.userType();
```

### 8.3 `api/common/GlobalExceptionHandler.java`

统一处理：

```text
BusinessException       → 业务错误
参数校验异常             → 参数错误
其他 Exception          → 系统内部错误
```

因此 Controller 一般不需要写 `try-catch`。

### 8.4 `api/config/`

`MybatisPlusConfig` 注册 MyBatis-Plus 分页插件。

`WebConfig` 配置：

- JWT 拦截器
- 跨域
- `/uploads/**` 文件访问路径

### 8.5 `api/controller/`

- `NewsController`：标准分层的新闻接口。
- `AuthController`：登录、注册、验证码和刷新 token。
- `UserController`：个人资料、头像上传和修改密码。
- `notice/course/score/exam`：通知与教务接口。
- `payment/card/dormitory/repair`：校园生活服务接口。
- `product/activity`：校园社区交易与活动接口。
- `location/assistant`：地图导航与智慧助手接口。

每个 Controller 只负责 HTTP 参数、调用对应 AppService、包装 `Result`；状态迁移和
计算规则必须放在 domain，事务用例放在 application。

### 8.6 `resources/application.yml`

包含：

- 服务端口
- MySQL 连接
- Redis 连接
- MyBatis-Plus 配置
- JWT 密钥
- 上传路径
- 日志级别

## 9. 从零创建一个标准新闻模块

下面假设项目里还没有新闻功能，从零建立一个完整模块。

### 9.1 第一步：设计数据库表

```sql
CREATE TABLE t_news (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,
    content LONGTEXT NOT NULL,
    category VARCHAR(100),
    cover_image VARCHAR(500),
    author VARCHAR(100),
    status VARCHAR(30) NOT NULL,
    view_count INT NOT NULL DEFAULT 0,
    publish_time DATETIME NULL,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
        ON UPDATE CURRENT_TIMESTAMP,
    deleted TINYINT NOT NULL DEFAULT 0
);
```

`deleted` 用于逻辑删除：

```text
0 = 未删除
1 = 已删除
```

### 9.2 第二步：创建 domain 文件

创建：

```text
campus-domain/
└── src/main/java/com/campus/domain/news/
    ├── News.java
    └── NewsRepository.java
```

#### `News.java`

```java
public class News {

    public enum Status {
        DRAFT, PUBLISHED, OFFLINE
    }

    private Long id;
    private String title;
    private Status status;
    private LocalDateTime publishTime;

    public void publish() {
        if (title == null || title.isBlank()) {
            throw new IllegalStateException("标题不能为空");
        }

        status = Status.PUBLISHED;
        publishTime = LocalDateTime.now();
    }

    public void offline() {
        status = Status.OFFLINE;
    }
}
```

#### `NewsRepository.java`

```java
public interface NewsRepository {
    News save(News news);
    Optional<News> findById(Long id);
    void deleteById(Long id);
}
```

此时还不能真正访问数据库，只是定义了业务模型和业务需要的存储能力。

### 9.3 第三步：创建 application 文件

创建：

```text
campus-application/
└── src/main/java/com/campus/application/news/
    ├── NewsAppService.java
    ├── command/
    │   └── CreateNewsCommand.java
    ├── query/
    │   └── NewsPageQuery.java
    └── dto/
        └── NewsDTO.java
```

#### Command

```java
public class CreateNewsCommand {

    @NotBlank
    private String title;

    @NotBlank
    private String content;

    @NotBlank
    private String author;
}
```

#### Query

```java
public class NewsPageQuery extends PageQuery {
    private String keyword;
    private String category;
}
```

#### DTO

```java
public class NewsDTO {
    private Long id;
    private String title;
    private String content;
    private Boolean published;
}
```

#### AppService

```java
@Service
public class NewsAppService {

    private final NewsRepository newsRepository;

    public NewsAppService(NewsRepository newsRepository) {
        this.newsRepository = newsRepository;
    }

    @Transactional
    public Long create(CreateNewsCommand command) {
        News news = new News();
        news.setTitle(command.getTitle());
        news.setContent(command.getContent());
        news.setAuthor(command.getAuthor());
        news.setViewCount(0);

        news.publish();

        return newsRepository.save(news).getId();
    }
}
```

这里注入的是 `NewsRepository` 接口，而不是 `NewsRepositoryImpl` 或 `NewsMapper`。

### 9.4 第四步：创建 infrastructure 文件

创建：

```text
campus-infrastructure/
└── src/main/java/com/campus/infrastructure/persistence/news/
    ├── NewsPO.java
    ├── NewsMapper.java
    ├── NewsConverter.java
    └── NewsRepositoryImpl.java
```

#### PO

```java
@TableName("t_news")
public class NewsPO {

    @TableId(type = IdType.AUTO)
    private Long id;

    private String title;
    private String content;
    private String status;

    @TableLogic
    private Integer deleted;
}
```

#### Mapper

```java
@Mapper
public interface NewsMapper extends BaseMapper<NewsPO> {
}
```

#### Converter

```java
public static News toDomain(NewsPO po) {
    News news = new News();
    news.setId(po.getId());
    news.setTitle(po.getTitle());
    news.setStatus(News.Status.valueOf(po.getStatus()));
    return news;
}
```

同时还需要实现反方向转换：

```java
public static NewsPO toPO(News news)
```

#### RepositoryImpl

```java
@Repository
public class NewsRepositoryImpl implements NewsRepository {

    private final NewsMapper mapper;

    public NewsRepositoryImpl(NewsMapper mapper) {
        this.mapper = mapper;
    }

    @Override
    public News save(News news) {
        NewsPO po = NewsConverter.toPO(news);

        if (po.getId() == null) {
            mapper.insert(po);
        } else {
            mapper.updateById(po);
        }

        news.setId(po.getId());
        return news;
    }
}
```

到这里，domain 要求的存储能力有了真实实现。

### 9.5 第五步：创建 API 文件

创建：

```text
campus-api/
└── src/main/java/com/campus/api/controller/news/
    └── NewsController.java
```

```java
@RestController
@RequestMapping("/api/news")
public class NewsController {

    private final NewsAppService newsAppService;

    public NewsController(NewsAppService newsAppService) {
        this.newsAppService = newsAppService;
    }

    @Authenticated
    @PostMapping
    public Result<Long> create(
            @Valid @RequestBody CreateNewsCommand command) {
        return Result.success(newsAppService.create(command));
    }
}
```

当前项目已有的新闻创建接口没有添加 `@Authenticated`。正式后台发布接口建议添加，并进一步校验管理员身份。

## 10. Spring 启动时如何装配模块

运行 `CampusApplication` 后，Spring 会扫描到：

```text
NewsController          @RestController
NewsAppService          @Service
NewsRepositoryImpl      @Repository
NewsMapper              @MapperScan
```

然后自动完成装配：

```text
NewsController
    │ 需要 NewsAppService
    ▼
NewsAppService
    │ 需要 NewsRepository
    ▼
NewsRepositoryImpl
    │ 需要 NewsMapper
    ▼
MyBatis 创建 NewsMapper 代理对象
```

虽然 `NewsAppService` 依赖的是接口：

```java
NewsRepository
```

但 Spring 能发现：

```java
NewsRepositoryImpl implements NewsRepository
```

所以最终会把 `NewsRepositoryImpl` 注入 `NewsAppService`。

## 11. 创建新闻的完整运行流程

前端发送：

```http
POST /api/news
Content-Type: application/json
Authorization: Bearer <token>
```

请求体：

```json
{
  "title": "校园运动会通知",
  "content": "本周五举办校园运动会",
  "author": "校办公室",
  "category": "通知公告"
}
```

完整调用过程：

```text
1. HTTP 请求进入 Spring MVC
             │
2. AuthInterceptor 检查 JWT
             │
3. JSON 转换为 CreateNewsCommand
             │
4. @Valid 校验标题、内容和作者
             │
5. NewsController.create(command)
             │
6. NewsAppService.create(command)
             │
7. new News() 创建领域实体
             │
8. news.publish() 执行业务规则
             │
9. NewsRepository.save(news)
             │
10. 实际进入 NewsRepositoryImpl
             │
11. NewsConverter.toPO(news)
             │
12. NewsMapper.insert(newsPO)
             │
13. MyBatis 执行 INSERT INTO t_news
             │
14. 数据库返回自增 ID
             │
15. NewsAppService 返回 ID
             │
16. Controller 使用 Result.success() 包装
             │
17. 返回 JSON
```

响应类似：

```json
{
  "code": 0,
  "msg": "成功",
  "data": 12
}
```

## 12. 本地运行步骤

### 12.1 准备环境

项目要求：

```text
JDK 17
Maven 建议 3.9.x
MySQL 8.x
```

检查命令：

```powershell
java -version
mvn -version
```

两个输出中的 Java 都应该是 17。

当前开发机终端实际检测到的是 Java 11 和 Maven 3.6.1，不满足项目的 Java 17 要求。

### 12.2 Maven 本地仓库问题

当前 Maven 全局配置把本地仓库指向：

```text
E:\java\apache-maven-3.6.1\meven_repo
```

该目录不存在，因此构建会在下载依赖前失败。

可以使用项目配置，并临时指定本地仓库：

```powershell
mvn -s docker/maven-settings.xml `
  -Dmaven.repo.local="$env:USERPROFILE\.m2\repository" `
  -DskipTests package -pl campus-api -am
```

更推荐安装 Maven 3.9.x，并删除旧 Maven 全局 `settings.xml` 中错误的配置：

```xml
<localRepository>E:\java\apache-maven-3.6.1\meven_repo</localRepository>
```

### 12.3 创建数据库

```sql
CREATE DATABASE campus_os
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

然后至少需要创建：

```text
t_news
t_campus_record
```

当前仓库没有 Flyway、Liquibase 或 SQL 初始化脚本，因此数据库和表不会自动创建。

### 12.4 配置数据库连接

默认配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/campus_os
    username: root
    password: root
```

如果本机数据库密码不是 `root`，需要修改 `campus-api/src/main/resources/application.yml`。

### 12.5 编译

在 `backend` 根目录执行：

```powershell
mvn clean package -DskipTests -pl campus-api -am
```

参数含义：

```text
-pl campus-api   构建 campus-api
-am              同时构建它依赖的其他模块
-DskipTests      暂时跳过测试
```

构建成功后生成：

```text
campus-api/target/campus-api.jar
```

### 12.6 启动

直接运行 JAR：

```powershell
java -jar campus-api/target/campus-api.jar
```

开发环境也可以使用：

```powershell
mvn -pl campus-api -am spring-boot:run
```

看到类似日志表示启动成功：

```text
Tomcat started on port 8080
Started CampusApplication
```

### 12.7 测试新闻列表

```powershell
Invoke-RestMethod `
  -Method Get `
  -Uri "http://localhost:8080/api/news?pageNum=1&pageSize=10"
```

### 12.8 登录并获取 token

```powershell
$login = Invoke-RestMethod `
```

## 13. 推荐阅读顺序

第一次阅读项目时，建议按照一次新闻详情请求的顺序阅读：

1. `NewsController`
2. `NewsAppService`
3. `News`
4. `NewsRepository`
5. `NewsRepositoryImpl`
6. `NewsConverter`
7. `NewsMapper`
8. `NewsPO`
9. `application.yml`

始终记住下面这条主线：

```text
Controller
   ↓
AppService
   ↓
领域实体 + Repository 接口
   ↓
RepositoryImpl
   ↓
Converter
   ↓
Mapper + PO
   ↓
MySQL
```

这就是当前项目中，一个标准模块从文件创建到请求执行和数据库落库的完整流程。
