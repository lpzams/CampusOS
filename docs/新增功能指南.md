# 新增功能指南（照着 news 模块抄一遍）

> 本项目的 **news（校园新闻）模块是全链路模板**：数据库 → 后端五层 → 网站端 → 小程序端。
> 加任何新功能，本质上就是"把 news 的每个文件照抄一遍、换成你的业务"。
> 下文以第 4 个功能模块「校园公告 **notice**」为例，你换成自己的功能名即可。

## 总览：一个功能要动哪些地方

| 端 | 要新建/修改的东西 | 参照模板 |
| --- | --- | --- |
| 数据库 | 建表 SQL | `docs/sql/001_init.sql` 里的 `t_news` |
| 后端 | domain / infrastructure / application / api 四层共 8 个类 | `backend/*/news/` 下同名文件 |
| 网站端 | api 文件 + 页面 + 路由（+ 顶部菜单） | `web/src/api/news.ts`、`web/src/views/news/` |
| 小程序端 | api 文件 + 页面 + pages.json 注册 | `miniapp/api/news.js`、`miniapp/pages/` |

后端依赖方向（内层不知道外层的存在，只能外层引内层）：

```
api  →  application  →  domain  ←  infrastructure
（Controller）（AppService/DTO） （实体/仓储接口）  （PO/Mapper/仓储实现）
```

## 第 0 步：设计并创建数据库表

1. 在 `docs/sql/` 下新建 `002_notice.sql`（编号递增，**不要改别人的文件**）；
2. 照抄 `001_init.sql` 里 `t_news` 的写法建 `t_notice`，注意四件事：
   - **文件开头两行照抄 001**：`SET NAMES utf8mb4;`（否则 Docker 自动执行时中文乱码）
     和 `USE campus_os;`（Docker 首次启动按文件名顺序自动执行这些脚本）；
   - 列名用下划线（`publish_time`），Java 字段用驼峰（`publishTime`），MyBatis-Plus 自动映射；
   - 保留三个"标配列"：`create_time`/`update_time`（数据库默认值维护，Java 不填）、`deleted`（逻辑删除）;
   - 主键 `id BIGINT AUTO_INCREMENT`。
3. 验证：`docker compose down -v && docker compose up -d` 重建数据库，确认你的表建出来了
   （用本机 MySQL 的同学手动执行你的脚本即可）。

## 第 1 步：后端 domain 层（业务核心，纯 Java）

| 新建文件 | 照抄 | 说明 |
| --- | --- | --- |
| `campus-domain/.../domain/notice/Notice.java` | `News.java` | 领域实体：字段 + 业务方法（如 `publish()`），**不带任何框架注解** |
| `campus-domain/.../domain/notice/NoticeRepository.java` | `NewsRepository.java` | 仓储接口：声明"我需要哪些存取能力"，实现在第 2 步 |

## 第 2 步：后端 infrastructure 层（数据库落地）

| 新建文件 | 照抄 | 说明 |
| --- | --- | --- |
| `campus-infrastructure/.../persistence/notice/NoticePO.java` | `NewsPO.java` | `@TableName("t_notice")`，字段对齐表列，带 `@TableLogic deleted` |
| `.../NoticeMapper.java` | `NewsMapper.java` | 继承 `BaseMapper<NoticePO>`，一般空接口即可 |
| `.../NoticeConverter.java` | `NewsConverter.java` | PO ↔ 领域实体 互转 |
| `.../NoticeRepositoryImpl.java` | `NewsRepositoryImpl.java` | `implements NoticeRepository`，用 Mapper 实现每个方法 |

## 第 3 步：后端 application 层（用例编排）

| 新建文件 | 照抄 | 说明 |
| --- | --- | --- |
| `campus-application/.../application/notice/dto/NoticeDTO.java` | `NewsDTO.java` | 返回给前端的数据结构 |
| `.../query/NoticePageQuery.java` | `NewsPageQuery.java` | `extends PageQuery`，只补自己的筛选字段 |
| `.../command/CreateNoticeCommand.java` | `CreateNewsCommand.java` | 写操作入参，`@NotBlank` 等校验写在这里 |
| `.../NoticeAppService.java` | `NewsAppService.java` | 编排：调实体方法 + 仓储；写方法加 `@Transactional` |

## 第 4 步：后端 api 层（HTTP 入口）

| 新建文件 | 照抄 | 说明 |
| --- | --- | --- |
| `campus-api/.../controller/notice/NoticeController.java` | `NewsController.java` | `@RequestMapping("/api/notice")`；只做"收参数 → 调 AppService → 包 `Result`"，**不写业务** |

写完启动 `CampusApplication`，用浏览器/Apifox 先把接口调通：
`GET http://localhost:8080/api/notice?pageNum=1&pageSize=10` 应返回 `{"code":0,...}`（code=0 即成功）。

## 第 5 步：网站端（web/）

1. `src/api/notice.ts`：照抄 `api/news.ts` —— 接口函数 + TS 类型（对着你的 DTO 写）；
2. `src/views/notice/` 下建页面：
   - 列表页照抄 `views/news/NewsListView.vue`（搜索 + 分页）；
   - 详情页照抄 `views/news/NewsDetailView.vue`（路由参数）；
   - 管理页照抄 `views/admin/NewsManageView.vue`（表格 + 弹窗表单 + 二次确认）；
3. `src/router/index.ts`：加路由；要进顶部导航就在 `App.vue` 的 `el-menu` 加一项。

## 第 6 步：小程序端（miniapp/）

1. `api/notice.js`：照抄 `api/news.js`；
2. `pages/notice/` 下建页面：列表照抄 `pages/index/index.vue`，详情照抄 `pages/news/detail.vue`；
3. **`pages.json` 里注册新页面**（漏了这步页面打不开）；
4. 从首页或其它页 `uni.navigateTo({ url: '/pages/notice/xxx' })` 跳转过去。

## 常见坑

| 坑 | 说明 |
| --- | --- |
| 前端接口 404 | 网站端开发环境必须走 `/api` 相对路径（vite 代理转发），别写 `http://localhost:8080` 绝对地址 |
| 查库字段全是 null | 表列名和 PO 字段没对上：列用下划线、字段用驼峰才能自动映射 |
| 删除后还查得出来 | 忘了在 PO 的 `deleted` 字段上加 `@TableLogic`（参照 NewsPO） |
| 小程序请求失败 | 微信开发者工具没勾「不校验合法域名」，或后端没启动；真机要把 BASE_URL 换成电脑局域网 IP |
| 分页不对 | 后端页码从 1 开始（`PageQuery`），前端别传 0 |
| Controller 里写业务 | 不允许。业务规则进领域实体，流程编排进 AppService，Controller 只做转发 |
