# 详细的操作步骤 —— 爬取「活水」老师评价

> 目标站点：https://app.huoshui.org/tabs/tab1 （活水，课程/老师评价平台）
> 要抓的东西：**对老师的评价**（老师名 + 课程 + 评价正文 + 评分 + 标签）
>
> 我已经把这个站点分析完了，结论写在 [schema.md](schema.md)，并配好了可直接运行的脚本。
> 下面分两部分：**A. 直接出数据（3 分钟）** 和 **B. 完整 HAR 工作流（学方法，下次自己搞）**。

# A. 直接出数据（最快路线）

因为「活水」的后端是 **LeanCloud REST API**，接口无需登录、无签名，我已经逆向清楚了，你不用自己抓包，直接跑脚本即可。

### 步骤 1：装依赖

```bash
cd web-crawler/scripts
pip install -r requirements.txt      # 只有一个 requests
```

### 步骤 2：先小批量验证（建议）

```bash
python scrape_reviews.py --limit 20
```

看到 `output/reviews.csv` 和 `output/reviews.json` 生成，用 Excel 打开 csv 能看到老师名/课程/评价，就说明通了。

### 步骤 3：抓全量老师评价（约 1.6 万条，1 分钟左右）

```bash
python scrape_reviews.py
```

结果：
- `output/reviews.json` —— 完整原始字段
- `output/reviews.csv` —— 拍平的表格（老师、课程、评价正文、评分、出勤、水课度、标签…），可直接用 Excel/pandas 分析

### 常用变体

```bash
python scrape_reviews.py --prof 何瑞文          # 只抓某位老师的全部评价
python scrape_reviews.py --classes Courses      # 抓课程聚合表（每位老师每门课的平均分）
python scrape_reviews.py --classes Reviews Courses   # 两张表都抓
python scrape_reviews.py --limit 200 --sleep 0.8     # 更慢更礼貌
```

> 字段含义、还能怎么筛（按分数、按院系等），见 [schema.md](schema.md) 第四节。

# B. 完整 HAR 工作流（学会方法，任何站点通用）

上面是"我已经帮你抓过包"的结果。如果你想**自己掌握这套方法**（下次换个网站也能干），按下面五步走。这也正是你发我那篇文章讲的流程。

## 第 1 步：清空缓存 + 配置 DevTools

用 Chrome/Edge 打开 https://app.huoshui.org/tabs/tab1 ，按 `F12` → **Network** 面板，必须做三件事：

1. ✅ 勾选 **`Preserve log`**（保留日志，页面跳转不丢包）
2. ✅ 勾选 **`Disable cache`**（禁用缓存，保证请求真实发出）
3. 点 🚫 **清空**当前记录，从零开始

要抓干净会话就用**无痕窗口**（`Ctrl+Shift+N`），或在 `Application → Clear storage → Clear site data` 清掉本站数据。

## 第 2 步：手动探索网站

**人来点，不要让 AI 点**。按你要复现的流程操作：

1. 进入首页/评价列表（tab1）
2. 往下滚动，触发"加载更多"
3. 点进某位老师/某门课，看详情和评价列表
4. （这个站不用登录就能看评价；若你要抓的功能需要登录，就把登录也走一遍）

小技巧：DevTools Network 顶部点 **`Fetch/XHR`** 过滤器，把图片/css 滤掉，肉眼确认真正的接口都出现了（你会看到 `lean.huoshui.org/1.1/classes/...` 这类请求）。

## 第 3 步：导出 HAR

Network 面板里右键任意请求 → **`Save all as HAR with content`**，保存到 `web-crawler/hars/`，语义化命名，比如：

```
hars/01-list.har        # 评价列表
hars/02-teacher.har     # 老师详情
```

> ⚠️ HAR 里含 Cookie/Session 等敏感信息，本目录已加进 `.gitignore`，不会被提交。

## 第 4 步：用工具 + AI 分析清洗 → schema.md

先用本工作区的 `analyze_har.py` 把噪音滤掉、把真接口提炼出来：

```bash
cd web-crawler/scripts

# 列出这份 HAR 里所有真接口（已自动过滤 css/图片/埋点）
python analyze_har.py ../hars/01-list.har

# 只看和评价相关的
python analyze_har.py ../hars/01-list.har --grep classes

# 看某一条的完整请求头+请求体+响应体
python analyze_har.py ../hars/01-list.har --detail 0

# 导出精简 JSON，整段复制给 AI 分析
python analyze_har.py ../hars/01-list.har --json
```

然后把 `--json` 的输出丢给 AI，配这样的提示词：

```
这是从某站点抓的 HAR 精简结果。请产出 schema.md：
1. 鉴权机制：靠哪些 header/参数？是否要登录？token 从哪来、怎么传递？
2. 请求模板：每个核心接口的 method / URL / 必需 header / 参数 / body。
3. 分页机制：靠 limit+skip 还是游标？最大页大小多少？
4. 时序依赖：后一个请求哪个参数来自前一个响应的哪个字段？
5. 过滤掉静态资源、埋点、心跳等无关请求。
```

产出的 `schema.md` 就是**可复现的核心资产**（本工作区已给你写好了一份现成的，见 [schema.md](schema.md)，可对照学习）。

## 第 5 步：根据 schema.md 生成脚本

有了干净的 schema，让 AI 生成脚本就很稳，提示词示例：

```
根据 schema.md，用 Python + requests 写一个爬虫：
- 分页抓取 Reviews 表全部数据（注意 skip>10000 不可靠，用 createdAt 游标翻页 + objectId 去重）
- 输出 json 和 csv，csv 拍平出：老师、课程、评价正文、评分、出勤、水课度、标签
- 每页之间限速 0.4s，加超时和错误处理
- 关键 key 写成常量放文件顶部
```

生成后**用 schema 逐条核对**（header 齐不齐、翻页对不对、字段来源对不对），跑通小批量再全量。
—— 本工作区的 [scripts/scrape_reviews.py](scripts/scrape_reviews.py) 就是这么产出并验证过的成品。

## 本站分析结论速查

| 项目 | 结论 |
|---|---|
| 前端 | Ionic + Angular SPA，数据全走 XHR |
| 后端 | LeanCloud REST API，`https://lean.huoshui.org/1.1` |
| 鉴权 | 请求头 `X-LC-Id` + `X-LC-Key`（网页公开下发的匿名 key），**读评价无需登录** |
| 评价数据 | `Reviews` 表，约 16573 条（老师名 `profName` + 评价 `comment` + 评分 `rating`…） |
| 聚合数据 | `Courses` 表，约 6600 条（每位老师每门课的平均分） |
| 反爬 | 无签名、无验证码、无时间戳校验 —— 最好爬的一类 |
| 翻页 | `limit`(≤1000) + `createdAt` 游标；全量约 17 个请求 |

细节全在 [schema.md](schema.md)。

## 合规与注意事项

- 数据是用户公开贡献的评价，**仅供个人分析研究**；不要二次公开发布、不要用于骚扰或攻击老师/作者。
- `authorId` 是匿名用户，**不要尝试反查真实身份**。
- 请**限速**，别并发猛冲给对方服务器添乱（脚本默认每页 0.4s 间隔，已经很轻量）。
- 只做只读查询，不做任何写入/删除操作。
- 抓包得到的 HAR 含敏感信息，别提交进仓库、别随意分享。
